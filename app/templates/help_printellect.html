🔌 First Time Setup 🎮 Device Controls 🔄 Reset & Recovery {% if is_admin %} ⚙️ Admin Guide {% endif %}
  1. 1

    Power on

    Switch on your Printellect base and wait for the LED to begin flashing. This signals it is in setup mode.

  2. 2

    Join PRINTELLECT-SETUP-xxxx

    From your phone or computer's Wi-Fi settings, join the setup network. The last 4 characters of the network name match the last 4 of your device ID (printed on the label).

  3. 3

    Open the local setup page

    In your browser, navigate to http://192.168.4.1/. You'll see a setup form — enter:

    • Your claim code (printed on the label / QR code)
    • Your home Wi-Fi network name (SSID)
    • Your home Wi-Fi password

    Tap Save and Reboot. The device will restart and connect to your home network.

  4. 4

    Claim in Printellect

    Come back to your home Wi-Fi. Go to Add Device and either scan the QR code on the label or manually enter your device ID and claim code. Give your device a friendly name (optional) and tap Claim Device.

  5. 5

    Confirm online

    Your device will provision its auth token automatically over the cloud. Within a minute it should appear as Online in My Devices. Tap it to open the control panel.

💡 Having trouble?

  • • Make sure you're on the PRINTELLECT-SETUP-xxxx network when opening the setup page.
  • • The claim code is case-sensitive. Double-check for zeros vs. letter O, etc.
  • • After saving Wi-Fi settings, wait ~30 seconds before claiming.
  • • If the device doesn't go online, try a 10-second reset (see Reset section below).

Playback — Play Perk / Stop

Select a perk theme from the dropdown and tap Play Perk to trigger the full light and audio experience on the device. Tap Stop to end immediately. The device queues the command — there may be a moment's delay.

🔊

Volume (0–30)

Drag the slider to set the speaker output level used during perk playback. Tap Apply to send. The slider automatically syncs back to the device's current setting every 3 seconds (unless you've just adjusted it).

💡

Brightness (0–100%)

Controls the LED strip intensity during both idle animations and active playback. Tap Apply to send. Reducing brightness extends LED life in always-on displays.

🌙

Idle Mode

Sets the LED animation shown when no perk is playing:

  • default — device-preset idle behavior
  • off — LEDs turn off when idle
  • ambient — soft, constant glow
  • pulse — slow breathing effect

Tap Set Idle Mode after choosing. The Reboot button performs a soft restart — useful if the device becomes unresponsive; settings are not lost.

🔬

Diagnostics

Light Test runs a temporary LED pattern for a set duration — useful for verifying LEDs work without a full perk play. Test Audio plays a short sample of the chosen track so you can verify the speaker output. Neither test affects your saved settings.

📊

Device Status

Shows the device's current playback state, what is playing (if anything), volume level, and when the device was last seen. Updates every 3 seconds while the page is open. Controls are automatically disabled when the device is offline.

3×

Triple press (within 3 seconds)

Soft reboot — restarts the firmware without clearing any settings, Wi-Fi credentials, or pairing token. Use this if the device becomes sluggish or stops responding.

10s

Hold 10 seconds — Wi-Fi reset

Clears saved Wi-Fi credentials so you can re-enter them via the setup network (PRINTELLECT-SETUP-xxxx). Your claim code and pairing token are preserved — you won't need to re-claim after reconnecting to Wi-Fi.

20s

Hold 20 seconds — Full factory reset / unpair

Clears all settings: Wi-Fi, pairing, and auth token. The device returns to AP setup mode. You will need to go through the full setup process again and re-claim a new claim code from your admin.

After any reset, the device returns to AP setup mode and broadcasts the PRINTELLECT-SETUP-xxxx network.

Provisioning a new device

  1. 1. Go to Admin → Hardware → Devices.
  2. 2. Click Create Device. The system auto-generates a device_id and claim_code — or enter your own.
  3. 3. Download device.json and the QR SVG from the label panel. Flash device.json to the Pico's filesystem before deployment.
  4. 4. Print the QR label sticker (2×2 inches) and attach it to the device housing. The QR encodes the pairing URL with the pre-filled device ID and claim code.
  5. 5. Hand the device to the end user. They follow the First Time Setup flow above.

OTA Firmware Updates

  1. 1. Go to Admin → Hardware → Releases.
  2. 2. Upload a new firmware release — either a zip bundle (preferred) containing all files, or a manifest + bundle pair. Set the version number and release channel (stable / beta).
  3. 3. Click Promote on the release you want to make current. This sets it as the latest target for OTA commands.
  4. 4. To push an update to a specific device, open the device's control page and find the Firmware Update section. Enter latest or a specific version and click Apply Update.
  5. 5. The device downloads and applies the update on its next polling cycle, then reboots. Monitor online status in Admin → Devices.
⚠️ If an OTA update fails, the device boots from its last-known-good image. Check the firmware zip contains valid main.py and manifest.json entries before promoting.

Unclaiming a device

In Admin → Devices, click Unclaim next to the device. This removes the user binding and auth token — the device must be factory reset (20-second hold) and re-provisioned before loaning to another user.
