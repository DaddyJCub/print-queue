v0.19.0 March 2026 ✨ Latest 🚀 Feature

Dashboard Home + Release Hardening

Introduced the dashboard-first user experience and hardened quote/checkout/account paths for production rollout.

🚀 Highlights

  • • Added a dashboard landing page at / with quick actions, active requests, queue insights, and printer status.
  • • Hardened admin quote and embedded checkout flows for unified account compatibility and cleaner rollout to production.

✨ Enhancements

  • • New landing page with greeting, 2×2 quick-action grid, active request cards, queue and fun stats, and announcements.
  • • Live printer status row with horizontal scroll, showing each printer's state, current file, and progress.
  • • Activity feed displaying recent queue events so users can see what's happening at a glance.
  • • The request form now lives on its own page, with navigation and buttons updated to match.
  • • Added startup env toggles to run credit auto-grants and USPS poller on a single primary replica only.

🐞 Bug Fixes

  • • Fixed admin quote-setting to safely handle SQLite row objects and avoid sync/async auth misuse in audit logging.
  • • Fixed embedded Stripe checkouts to resolve the authenticated accounts.id for both legacy and unified sessions.
  • • Fixed credit checkouts to resolve or create unified account mappings instead of using legacy user IDs.

📝 Notes / Things to Know

  • • Dashboard rollout is controlled by the dashboard_home feature flag.
  • • For multi-replica deploys, set ENABLE_CREDIT_GRANT_SCHEDULER and ENABLE_USPS_TRACKING_POLLER to true on only one replica.
