# Print Agent Versioning Policy

This repository uses manual semantic versioning for the print agent.

## Scope

This policy applies to all changes under:

- agent/printqueue_agent/

## Required Rules

- Bump the print agent version whenever behavior changes in the print agent.
- Update version only in: agent/printqueue_agent/__init__.py
- Use plain semantic versions only: MAJOR.MINOR.PATCH
- Do not use autogenerated build suffixes (for example: +auto.1234567890).
- Keep versions monotonic (never decrease).

## Bump Guidance

- PATCH: bug fixes, stability improvements, non-breaking internal changes.
- MINOR: new features or non-breaking behavior additions.
- MAJOR: breaking behavior or compatibility changes.

## Examples

- Valid: 1.1.0, 1.1.1, 1.2.0, 2.0.0
- Invalid: 1.1.0+auto.1720826312

## Enforcement Intent

The goal is deterministic OTA/update behavior and clear release tracking.

Any contributor or automation touching print-agent behavior must follow this file.
